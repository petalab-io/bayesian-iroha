# Webサイト高速化施策の ROI 評価レポート

## 1. ビジネス課題の定義
Webサイトの表示速度（LCP: Largest Contentful Paint）改善施策が、単なる技術的な向上に留まらず、**最終的な事業利益（ROI）にどれだけ寄与するか**を検証します。

特に、サイト全体の平均値改善に隠れた「ページごとの感度の違い」と「改悪リスク」を可視化し、根拠のあるリリース判定を行うことを目的とします。

## 2. 意思決定フレームワーク（3層フィルタリング）
本プロジェクトでは、以下の3段階の基準を用いて施策の成否を判定します。

1.  **HDI (統計的実証)**: 速度改善（差分）の 94% HDI が 0 を跨がず、統計的に有意な変化があるか。
2.  **ROPE (実質的等価性)**: その変化が、ユーザーが体感できる閾値（例: ±50ms）を超えているか。
3.  **ROI (経済的評価)**: 速度変化、ページビュー（PV）、売上感度、客単価を統合し、最終的な利益分布がプラスか。

## 3. 分析結果サマリー
階層ベイズモデルと感度分析（売上感度の不確実性を考慮）によるシミュレーション結果は以下の通りです。

| 指標 | 結果 | 評価 |
| :--- | :--- | :--- |
| **期待純利益 (Expected Net Profit)** | **約 8,210 万円** | 🟢 期待値としては大幅な黒字 |
| **収益化確率 (Win Rate)** | **65.1%** | 🟡 約35%の赤字リスクを内在 |
| **Checkout 損失リスク** | **77.4%** | 🔴 決済ページでの損失発生が極めて濃厚 |

### 意思決定のポイント
全体の利益期待値は高いものの、**「決済ページ（Checkout）における致命的なパフォーマンス劣化」**が判明しました。これは、Top/Detail ページの改善による利益を大きく棄損するリスク（隠れたリスク）を意味します。

## 4. 最終的な推奨アクション

> **判定：条件付きリリース（部分マージを推奨）**

1.  **Top / Detail ページの改善のみを先行リリース**:
    * 改善効果が確実視されており、リスクが低いため、この2ページに関する変更のみを本番環境へ適用します。
2.  **Checkout ページの変更をロールバック・再調査**:
    * 100%近い確率で損失を招くため、原因（エンジニアリング上のボトルネック）を特定し、修正が完了するまでリリースを凍結します。
3.  **モニタリングの継続**:
    * 部分リリース後、実数値としてのCVRおよびLCPの推移を1週間監視し、想定通りのROIが得られているか再評価します。
